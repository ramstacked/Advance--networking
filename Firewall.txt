# 🟢 STEP 1 – Understand Firewall Basics

Even before touching labs, you must **know the theory**:

### 🔹 Firewall Types

1. **Packet Filtering Firewall** – Blocks/Allows traffic by IP, port, protocol
2. **Stateful Firewall** – Tracks connection states
3. **Next-Generation Firewall (NGFW)** – Inspects applications, content, users
4. **Proxy Firewall** – Acts as an intermediary
5. **Web Application Firewall (WAF)** – Protects web apps

### 🔹 Key Concepts

* **Zones**: Inside, Outside, DMZ
* **Policies**: Allow/deny rules
* **NAT**: Translate internal IPs to public IPs
* **VPN**: Secure remote access
* **Logging/Monitoring**: Detect attacks

💡 Hint: Focus on **real-world firewall scenarios** like “allow web traffic, block all else” or “internal users → Internet, external users → block”.

# 🟠 STEP 3 – Free Lab Setup Guide

### Lab Environment:

* **PC** with 8–16GB RAM
* **GNS3 installed**
* **VirtualBox** or **VMware Player** for firewall VMs

### Suggested Lab Topology:

```
[Internet VM] --- [Firewall VM] --- [Internal LAN VM]
```

* Internet VM: Ubuntu/Debian
* Firewall VM: pfSense/OPNSense
* Internal LAN VM: Windows/Linux clients

### Key Configurations to Practice:

1. **Basic Firewall Rule**: Allow HTTP/HTTPS, block everything else
2. **NAT**: Internal LAN → Internet
3. **Port Forwarding**: Access internal web server from outside
4. **VPN**: Remote client connects via SSL VPN/IPSec
5. **DMZ**: Place a server in DMZ, allow limited access
6. **Logging & Alerts**: Check blocked traffic in dashboard
7. **IDS/IPS**: Install Snort/Suricata, simulate attacks

💡 Hint: Start small with **1 firewall + 1 LAN + Internet VM**, then expand with **2 LANs + DMZ + VPN**.

# 🔴 STEP 4 – Advanced Firewall Concepts

1. **Zone-Based Firewall** – Assign interfaces to zones, create zone-pair policies
2. **Advanced NAT** – Source NAT, Destination NAT, PAT, 1:1 NAT
3. **VPN** – Site-to-Site IPSec, Remote SSL VPN
4. **Application Layer Filtering** – Allow/block by app/protocol
5. **Firewall High Availability (HA)** – Active/Passive failover
6. **Logging & SIEM Integration** – Export logs to ELK/Graylog

💡 Hint: In labs, **practice both CLI (VyOS/Cisco ASA) and GUI (pfSense/OPNSense)**. Employers value both.
