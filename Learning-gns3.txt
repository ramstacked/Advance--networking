# 🔴 PHASE 4 – Network Security Basics

### ACLs (Access Control List)

* Standard ACL: filter source IP only

```bash
access-list 1 permit 192.168.1.0 0.0.0.255
interface g0/0
ip access-group 1 in
```

* Extended ACL: filter IP + protocol + port

```bash
access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80
```

### NAT (Network Address Translation)

* Static NAT: one-to-one
* PAT / Overload: many-to-one using public IP

### DHCP Server

* Let router assign IP automatically to devices

💡 Hint: Labs should simulate **Internet access for internal users** + **ACL filtering malicious traffic**

# 🟣 PHASE 5 – Firewalls (Job-Oriented)

### 1️⃣ Cisco ASA Firewall (Basic Commands)

* Basic interface: `nameif`, `security-level`, `ip address`
* Access rules: `access-list` + `access-group`
* NAT configuration: `nat (inside,outside)`

### 2️⃣ Zone-Based Firewall (Cisco IOS)

* Create zones: `zone security INSIDE`, `zone security OUTSIDE`
* Assign interfaces to zones
* Define policies: `zone-pair security INSIDE-TO-OUTSIDE`, `service-policy`

### 3️⃣ Fortinet / Palo Alto (GNS3 VM)

* Drag VM firewall into GNS3
* Configure internal/external interfaces
* Create policies for HTTP/HTTPS/SSH
* Enable NAT & logging

💡 Hint: Focus on **basic firewall policies**: allow inside to outside, block specific ports, NAT.
